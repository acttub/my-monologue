# 오래된 독백 — 작업 지침

> 이 저장소는 **공개**입니다. 비밀값·실명·비공개 경로·내부 티켓 링크를 커밋하지 마세요.

## 무엇인가

두 가지 선택(어떤 말을 하고 싶은지 · 길이)으로 **저작권이 끝난 근대 희곡의 독백**을 골라주는 미니 서비스. acttub(질문 기반 AI 연기 연습 파트너)의 유입용 서브프로젝트입니다.

**판정하지 않습니다.** 점수·등급·연기력 평가가 없고, 산출물은 대본뿐입니다.

## 이 저장소에서 가장 중요한 것: 런타임에 LLM이 없다

```
사용자 선택 → monologues.json 필터 → 하나 표시
```

서버 호출도, LLM도, API 키도 없습니다. **완전 정적 사이트입니다.**

### 왜 이렇게 됐나 — 이 결정을 되돌리지 마세요

이 서비스는 원래 **AI가 오리지널 독백을 생성**하는 방식이었고, 배포 전 안전 감사를 **9회** 통과하지 못했습니다. 위반 추이는 5 → 12 → 1 → 1 → 2 → 1 → 2 → 7이었고, 한 번도 0에 닿지 못했습니다.

매 회차 위반이 **새로운 형태**로 나타났습니다:

| 회차 | 형태 | 위험이 있던 층 |
|---|---|---|
| 1차 | "차라리 저를 데려가지" | 직접 진술 |
| 3차 | "당신의 선택", "삶의 뒷문" | 완곡어 |
| 4차 | "그저 모든 것을 멈춰버리고 싶습니다" | 대상 없는 소망 |
| 5차 | "스크린도어를 보며 멈추어 서는 연습" | 장면·장소 |
| 6차 | "당신과 함께 쉬고 싶습니다" | 구문 |
| 7차 | "명분을 간신히 붙들고 있습니다" | 문장 간 논리 |
| 8차 | 안전한 상쇄절이 마지막에 놓이지 않음 | 문단 배치 |
| 9차 | 상쇄절이 검사 창 바로 바깥으로 밀림 | **방어 규칙이 만든 경계** |

9차가 결정적이었습니다. "마지막 두 문장"이라는 경계를 코드로 긋자 위반이 **세 번째 문장으로 정렬**됐습니다. 규칙을 정교화할수록 우회가 정교해졌습니다.

**근본 원인은 런타임에 새 텍스트가 만들어진다는 것이었습니다.** 무한한 출력을 유한한 규칙으로 막으려 한 것입니다.

큐레이션은 그 전제를 없앱니다:
- 텍스트가 **유한하고 고정**이라 전수 검토가 가능합니다 (73편)
- 한 번 통과한 판정이 **영구히 유효**합니다 — 새 텍스트가 생기지 않으니까요
- 런타임 지연·비용·프롬프트 인젝션이 전부 **0**입니다

**그러니 "AI로 독백을 만들어주면 더 좋지 않나"는 제안이 오면 위 표를 보여주세요.** 이미 해봤고, 9번 실패했습니다.

## 데이터

`monologues.json` — 73편. 손으로 고치지 말고 파이프라인으로 재생성하세요.

```bash
node tools/extract.mjs > tools/extracted.json     # 위키문헌에서 원문 추출
GEMINI_API_KEY=... node tools/classify.mjs        # 분류 + 안전 선별 → monologues.json
```

**LLM은 여기서만 돕니다.** `classify.mjs`가 각 독백을 분류하고 안전 판정을 하며, 통과분만 `monologues.json`에 남깁니다. 실제로 17편이 안전 사유로 탈락했습니다 — 1910년대 작품이라 자살 소망("죽고 싶은 때도 한두 번이 아니언마는"), 자해 표현, 여성 비하 대사가 실재합니다.

**데이터를 바꾸면 반드시 전수 안전 재검토를 하세요.** 73편은 사람이 다 읽을 수 있는 규모이고, 그게 이 설계의 핵심입니다.

원문 출처는 전부 [위키문헌](https://ko.wikisource.org)이고 각 항목에 `source` URL이 있습니다.

**판정 규칙은 하나입니다: 작가와 번역자가 둘 다 1962년 이전에 사망했는가.** 2013년 저작권법 개정의 경과규정 때문입니다 — 그 이전에 이미 소멸한 저작권은 되살아나지 않아 사후 50년이 적용됩니다.

⚠️ **원작이 퍼블릭 도메인이어도 한국어 번역본은 번역자에게 별도 저작권이 있습니다**(저작권법 제5조). 민음사·열린책들 등 현대 번역본을 쓰면 침해입니다. 새 작품을 추가할 때 **번역자 사망연도를 반드시 확인**하세요.

⚠️ **유치진(1974 몰)은 2044년까지 저작권이 살아 있습니다.** 근대 희곡이라고 다 되는 게 아닙니다.

미해결 자문 항목 (법률 멘토 확인 필요): PD 저작물의 발췌·현대어 윤문이 저작인격권(제14조 제2항, 유족 청구권)에 걸리는 범위.

## 절대 금지 (사용자 노출 표면)

- 점수·등급·랭킹·연기력 판정
- 읽는 사람(배우)의 성격·역량·역할 적합성 서술 — "당신에게 어울리는"
- "이렇게 하세요" 식 교정·처방. **`situation` 필드에 연기 지시를 넣는 것이 여기 해당합니다**
- 감정 전달력·몰입도·자연스러움·진정성 판단, 관객 반응 단정

### `situation` 필드의 경계

인물이 처한 사정과 상대만 적습니다. 배우가 무엇을 어떻게 할지는 적지 않습니다.

| 허용 | 금지 |
|---|---|
| "남편에게 버림받고 시댁에 남겨진 여인이 시누이에게 속을 털어놓는다" | "슬픔을 억누르며 담담하게 말한다" |

## 카피

사용자에게 보이는 고정 텍스트는 전부 `copy.js`에 있습니다. HTML이나 JS에 문자열을 직접 박지 마세요. `monologues.json`의 `body`는 퍼블릭 도메인 원문이고, 그 밖의 모든 문장은 `copy.js`에서 나옵니다.

**원문 표기를 현대어로 고치지 마세요.** 1912~1934년 표기를 그대로 두고 랜딩에 안내 문구를 둡니다. 윤문은 위 저작인격권 이슈가 붙고, 무엇보다 원문의 질감이 이 서비스의 성격입니다.

## 구조

```
index.html        화면 4개(랜딩·선택·로딩·결과)
style.css         브랜드 토큰: 블루 #0A79FB, 밝은 중성 배경, Pretendard
app.js            선택 상태, 필터, 결과 렌더, 공유 카드 canvas
copy.js           확정 카피 — 여기 밖에 사용자 노출 문자열을 두지 않는다
monologues.json   독백 73편 (빌드 산출물)
tools/extract.mjs   위키문헌 → extracted.json
tools/classify.mjs  extracted.json → monologues.json (LLM은 여기서만)
```

**빌드 단계도 서버리스 함수도 없습니다.** 번들러·프레임워크를 도입하지 마세요.

## 배포

GitHub가 정본이고 Vercel git 연동이 걸려 있습니다. `git push origin main` 하면 자동 배포됩니다. 로컬에서 `vercel --prod`를 치지 않습니다.

**환경변수가 필요 없습니다.** 런타임에 외부 호출이 없습니다. `GEMINI_API_KEY`는 데이터 재생성 시 로컬에서만 씁니다.

## 배포 전 확인

데이터를 바꿨을 때만 필요합니다. 코드만 고쳤다면 생략해도 됩니다.

1. `monologues.json`의 모든 항목에 `source`·`license`가 있는가
2. 새로 추가된 작품의 작가·번역자가 둘 다 1962년 이전 사망인가
3. **새 텍스트 전수 안전 검토** — 자살·자해, 성적 내용, 차별 표현의 옹호
4. `situation` 필드에 연기 지시가 섞이지 않았는가
5. 선택 조합(target 5 × length 2 = 10칸)이 전부 채워져 있는가
